# Adopt the Playbook in an Existing Repository

## Manual Steps

1. Review the target repository README, branch protections, and existing contributor docs.
2. Copy or reference the baseline files: [`AGENTS.md`](../AGENTS.md), [`CLAUDE.md`](../CLAUDE.md), [`REVIEW_STANDARD.md`](../REVIEW_STANDARD.md), and [`EXECUTION_STANDARD.md`](../EXECUTION_STANDARD.md).
3. Update the target repository README with a short section naming this repository as the engineering source of truth.
4. Create or align the repository project board using [`../GITHUB_PROJECT_STANDARD.md`](../GITHUB_PROJECT_STANDARD.md).
5. Add issue templates, PR template, and any reusable workflow files that fit the target repository.
6. Record the adopted playbook version from [`../VERSION.md`](../VERSION.md).
7. Open a pull request summarizing files copied, files referenced, and any intentional deviations.
